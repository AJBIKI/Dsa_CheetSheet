The Mentor's Mindset

Before a single line of code — understand how to learn DSA correctly.

The Secret Nobody Tells You

DSA is not about memorizing solutions. It's about recognizing patterns. When you see a new problem, your brain should say "ah, this smells like a sliding window" — not "I've seen this exact problem before." That's the goal. Pattern recognition over rote memorization.

1

Understand Before You Code

Spend 10 minutes on a problem before touching the keyboard. Draw it out. What are the inputs/outputs? What's the brute force? What makes it slow?

2

Learn One Pattern at a Time

Do 5–8 problems on the same pattern before moving on. Depth beats breadth. You don't need 500 problems — you need 50 deeply understood ones.

3

Code Without Looking

After learning a pattern, close this guide and write the template from scratch. Then open a problem and solve it. Struggle is how the brain forms lasting memory.

4

Analyze Your Solutions

Every solution: What's the time complexity? Space complexity? Is there a better approach? Can you do it with O(1) space? Asking these questions is what separates good coders from great ones.

5

Revisit and Repeat

Come back to old problems after 1 week. If you can solve them in under 10 minutes, you've truly learned it. If not, redo them. Spaced repetition is the key.

Complexity Analysis (Big-O)

The language of algorithm efficiency. You must speak it fluently.

NotationNameExample10 elements1000 elementsVerdict
O(1)ConstantArray index access11✦ Perfect
O(log n)LogarithmicBinary search310✦ Excellent
O(n)LinearLoop through array101,000✦ Good
O(n log n)LinearithmicMerge sort3310,000◆ Acceptable
O(n²)QuadraticNested loops1001,000,000◆ Avoid if possible
O(2ⁿ)ExponentialNaive recursion1,02410³⁰¹✗ Dangerous
O(n!)FactorialAll permutations3,628,800Impossible✗ Avoid

How to Calculate Big-O — 3 Rules

  • Drop constants: O(2n) → O(n), O(100) → O(1)
  • Drop lower terms: O(n² + n) → O(n²)
  • Nested loops multiply: two nested loops over n = O(n²). One loop inside another that goes log n = O(n log n)
big_o_examples.pyPython
# O(1) — constant: doesn't grow with input
def get_first(arr):
    return arr[0]

# O(n) — linear: one loop
def find_max(arr):
    max_val = arr[0]
    for x in arr:
        if x > max_val:
            max_val = x
    return max_val

# O(n²) — quadratic: nested loops
def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        for j in range(0, n - i - 1):
            if arr[j] > arr[j + 1]:
                arr[j], arr[j + 1] = arr[j + 1], arr[j]

# O(log n) — binary search halves the search space each time
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target: return mid
        elif arr[mid] < target: lo = mid + 1
        else: hi = mid - 1
    return -1

  • Need O(1) membership checks? Use a set, not a list.
  • Need counts? Use Counter, not a manual dict.
  • Need index + value? Always use enumerate.
  • Modifying while iterating? Iterate a copy: for x in arr[:]

Linked Lists

Nodes connected by pointers. Core for understanding memory and pointers.

linked_list.pyPython
# ── NODE DEFINITION ──
class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

# ── TRAVERSAL ──
curr = head
while curr:
    print(curr.val)
    curr = curr.next

# ── REVERSE A LINKED LIST (must memorize) ──
def reverse_list(head):
    prev, curr = None, head
    while curr:
        nxt = curr.next     # save next
        curr.next = prev    # reverse pointer
        prev = curr         # move prev forward
        curr = nxt          # move curr forward
    return prev

# ── FIND MIDDLE (slow & fast pointer) ──
def find_middle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
    return slow

# ── DETECT CYCLE (Floyd's algorithm) ──
def has_cycle(head):
    slow = fast = head
    while fast and fast.next:
        slow = slow.next
        fast = fast.next.next
        if slow == fast:
            return True
    return False

# ── MERGE TWO SORTED LISTS ──
def merge_sorted(l1, l2):
    dummy = ListNode(0)
    curr = dummy
    while l1 and l2:
        if l1.val <= l2.val:
            curr.next = l1
            l1 = l1.next
        else:
            curr.next = l2
            l2 = l2.next
        curr = curr.next
    curr.next = l1 or l2   # attach remaining
    return dummy.next

# ── DUMMY NODE TRICK ──
# Use a dummy head to simplify edge cases
# (empty list, deleting head, etc.)
dummy = ListNode(0)
dummy.next = head
# ... work with dummy.next as the real head

⚠ Common Pitfall

Always save nxt = curr.next before modifying curr.next, or you'll lose the rest of the list. Also, use a dummy node — it eliminates 90% of edge cases in linked list problems.

stack = []

from collections import deque
q = deque()

q.popleft()        # dequeue  O(1) ← list.pop(0) is O(n)!
q[0]               # front peek
not q              # empty check

# ── MONOTONIC STACK (very useful!) ──
# Used for: next greater element, trap rain water, etc.
# Maintains elements in increasing or decreasing order

# Next Greater Element to the right
def next_greater(nums):
    result = [-1] * len(nums)
    stack = []  # stores indices
    for i, val in enumerate(nums):
        while stack and nums[stack[-1]] < val:
            idx = stack.pop()
            result[idx] = val
        stack.append(i)
    return result

# ── VALID PARENTHESES (classic stack problem) ──
def is_valid(s):
    stack = []
    pairs = {')': '(', '}': '{', ']': '['}
    for ch in s:
        if ch in '({[':
            stack.append(ch)
        elif not stack or stack[-1] != pairs[ch]:
            return False
        else:
            stack.pop()
    return not stack

def two_sum(nums, target):
    seen = {}  # val → index
    for i, x in enumerate(nums):
        need = target - x
        if need in seen:
            return [seen[need], i]
        seen[x] = i

def group_anagrams(strs):
    groups = defaultdict(list)
    for s in strs:
        key = "".join(sorted(s))  # sorted string as key
        groups[key].append(s)
    return list(groups.values())

max_heap = []
heapq.heappush(max_heap, -5)
heapq.heappush(max_heap, -3)
largest = -heapq.heappop(max_heap)  # 5

# ── HEAP WITH TUPLES ──

tasks = []
heapq.heappush(tasks, (2, "low priority"))
heapq.heappush(tasks, (1, "high priority"))

# ── K LARGEST ELEMENTS ──
def k_largest(nums, k):
    return heapq.nlargest(k, nums)   # O(n log k)

# ── TOP K FREQUENT ELEMENTS ──
def top_k_frequent(nums, k):
    freq = Counter(nums)
    return heapq.nlargest(k, freq.keys(), key=freq.get)

Trees & Binary Search Trees

Hierarchical data. Mastering trees unlocks a huge set of problems.

trees.pyPython
# ── NODE ──
class TreeNode:
    def __init__(self, val=0, left=None, right=None):
        self.val = val
        self.left = left
        self.right = right

# ── TRAVERSALS (learn all 4, internalize them!) ──

# Inorder: Left → Root → Right  (gives sorted order in BST!)
def inorder(root):
    if not root: return
    inorder(root.left)
    print(root.val)
    inorder(root.right)

# Preorder: Root → Left → Right  (useful for copying/serializing)
def preorder(root):
    if not root: return
    print(root.val)
    preorder(root.left)
    preorder(root.right)

# Postorder: Left → Right → Root  (useful for deletion)
def postorder(root):
    if not root: return
    postorder(root.left)
    postorder(root.right)
    print(root.val)

# Level Order (BFS) — uses a queue
from collections import deque
def level_order(root):
    if not root: return
    q = deque([root])
    result = []
    while q:
        level = []
        for _ in range(len(q)):  # process one level at a time
            node = q.popleft()
            level.append(node.val)
            if node.left:  q.append(node.left)
            if node.right: q.append(node.right)
        result.append(level)
    return result

# ── HEIGHT OF TREE ──
def height(root):
    if not root: return 0
    return 1 + max(height(root.left), height(root.right))

# ── BST OPERATIONS ──
# Search: go left if target < node, right if target > node
def bst_search(root, target):
    if not root or root.val == target: return root
    if target < root.val: return bst_search(root.left, target)
    return bst_search(root.right, target)

# Validate BST
def is_valid_bst(root, lo=float('-inf'), hi=float('inf')):
    if not root: return True
    if not (lo < root.val < hi): return False
    return (is_valid_bst(root.left, lo, root.val) and
            is_valid_bst(root.right, root.val, hi))

Graphs

The most expressive data structure. Maps to real-world problems directly.

graphs.pyPython
from collections import defaultdict, deque

# ── REPRESENTATION ──
# Adjacency list (most common for interviews)
graph = defaultdict(list)
graph[0].append(1)
graph[0].append(2)
graph[1].append(3)

# Build from edge list
edges = [(0,1), (0,2), (1,3)]
for u, v in edges:
    graph[u].append(v)
    graph[v].append(u)   # omit for directed graph

# ── DFS (Depth First Search) ──
# Recursive version
def dfs(node, visited, graph):
    visited.add(node)
    for nei in graph[node]:
        if nei not in visited:
            dfs(nei, visited, graph)

# Iterative version (avoids recursion limit)
def dfs_iter(start, graph):
    visited = set([start])
    stack = [start]
    while stack:
        node = stack.pop()
        for nei in graph[node]:
            if nei not in visited:
                visited.add(nei)
                stack.append(nei)

# ── BFS (Breadth First Search) ──
# Finds SHORTEST PATH in unweighted graph
def bfs(start, graph):
    visited = set([start])
    q = deque([start])
    dist = {start: 0}
    while q:
        node = q.popleft()
        for nei in graph[node]:
            if nei not in visited:
                visited.add(nei)
                dist[nei] = dist[node] + 1
                q.append(nei)
    return dist

# ── MATRIX DFS (very common!) ──
def num_islands(grid):
    if not grid: return 0
    rows, cols = len(grid), len(grid[0])
    visited = set()

    def dfs(r, c):
        if (r, c) in visited: return
        if not (0 <= r < rows and 0 <= c < cols): return
        if grid[r][c] == '0': return
        visited.add((r, c))
        for dr, dc in [(1,0),(-1,0),(0,1),(0,-1)]:
            dfs(r + dr, c + dc)

    count = 0
    for r in range(rows):
        for c in range(cols):
            if grid[r][c] == '1' and (r, c) not in visited:
                dfs(r, c)
                count += 1
    return count

Tries (Prefix Trees)

The go-to structure for anything involving word prefixes or autocomplete.

trie.pyPython
class TrieNode:
    def __init__(self):
        self.children = {}
        self.is_end = False

class Trie:
    def __init__(self):
        self.root = TrieNode()

    def insert(self, word):
        node = self.root
        for ch in word:
            if ch not in node.children:
                node.children[ch] = TrieNode()
            node = node.children[ch]
        node.is_end = True

    def search(self, word):
        node = self.root
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.is_end

    def starts_with(self, prefix):
        node = self.root
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True

# ── PYTHON BUILT-IN (use this in interviews) ──
arr.sort()                              # in-place, O(n log n)
arr.sort(reverse=True)               # descending
arr.sort(key=lambda x: x[1])          # sort by 2nd element
arr.sort(key=lambda x: (x[1], -x[0])) # multi-key sort
sorted(arr)                            # returns new list
sorted(arr, key=abs)                  # sort by absolute value

# ── MERGE SORT (divide & conquer — O(n log n) guaranteed) ──
def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    mid = len(arr) // 2
    left  = merge_sort(arr[:mid])       # sort left half
    right = merge_sort(arr[mid:])       # sort right half
    return merge(left, right)           # merge sorted halves

def merge(left, right):
    result = []
    i = j = 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i]); i += 1
        else:
            result.append(right[j]); j += 1
    result.extend(left[i:])              # attach remaining
    result.extend(right[j:])
    return result

# ── QUICK SORT (fast in practice, worst case O(n²)) ──
def quick_sort(arr, lo=0, hi=None):
    if hi is None: hi = len(arr) - 1
    if lo >= hi: return
    pivot_idx = partition(arr, lo, hi)
    quick_sort(arr, lo, pivot_idx - 1)
    quick_sort(arr, pivot_idx + 1, hi)

def partition(arr, lo, hi):
    pivot = arr[hi]
    i = lo - 1
    for j in range(lo, hi):
        if arr[j] <= pivot:
            i += 1
            arr[i], arr[j] = arr[j], arr[i]
    arr[i + 1], arr[hi] = arr[hi], arr[i + 1]
    return i + 1

# ── INSERTION SORT (best for nearly-sorted or tiny arrays) ──
def insertion_sort(arr):
    for i in range(1, len(arr)):
        key = arr[i]
        j = i - 1
        while j >= 0 and arr[j] > key:
            arr[j + 1] = arr[j]           # shift right
            j -= 1
        arr[j + 1] = key                  # place key

# ── COUNTING SORT (O(n+k) — for small integer ranges) ──
def counting_sort(arr):
    if not arr: return arr
    max_val = max(arr)
    count = [0] * (max_val + 1)
    for x in arr: count[x] += 1
    result = []
    for val, freq in enumerate(count):
        result.extend([val] * freq)
    return result

Which Sorting Algorithm to Pick?

  • Default: Always use Python's sort() / sorted() — Timsort is optimal.
  • Custom comparison: Use functools.cmp_to_key for complex orderings.
  • Nearly sorted data: Insertion Sort shines here — O(n) best case.
  • Small integer range: Counting Sort gives O(n) — beats comparison sorts.
  • Interview theory question: Know Merge Sort (stable, O(n log n) guaranteed) and why Quick Sort is faster in practice.

Binary Search

More than just searching a sorted array — it's a whole problem-solving paradigm.

The Key Insight

Binary search works on any monotonic condition — not just sorted arrays. "Find the minimum X such that condition(X) is true" — if you can check the condition in O(1) or O(n), binary search gets you to O(log n) or O(n log n).

binary_search.pyPython
# ── TEMPLATE 1: Standard Binary Search ──
def binary_search(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2          # no overflow in Python
        if arr[mid] == target: return mid
        elif arr[mid] < target: lo = mid + 1
        else: hi = mid - 1
    return -1

# ── TEMPLATE 2: Find Leftmost Position ──
# "First position where condition is true"
def lower_bound(arr, target):
    lo, hi = 0, len(arr)     # note: hi = len(arr)
    while lo < hi:           # note: strict <
        mid = (lo + hi) // 2
        if arr[mid] < target: lo = mid + 1
        else: hi = mid
    return lo               # lo == hi, insertion point

# ── TEMPLATE 3: Binary Search on Answer ──
# "Find minimum X where condition(X) is satisfiable"
def search_on_answer(lo, hi):
    def condition(x): ...   # returns True/False
    while lo < hi:
        mid = (lo + hi) // 2
        if condition(mid): hi = mid
        else: lo = mid + 1
    return lo

# ── BUILTIN BISECT ──
import bisect
bisect.bisect_left(arr, x)   # first index >= x
bisect.bisect_right(arr, x)  # first index > x
bisect.insort(arr, x)        # insert maintaining order

Two Pointers

Eliminate the inner loop. Turn O(n²) into O(n) on sorted arrays.

two_pointers.pyPython
# ── OPPOSITE ENDS ── (sorted array, find pair)
def two_sum_sorted(arr, target):
    lo, hi = 0, len(arr) - 1
    while lo < hi:
        s = arr[lo] + arr[hi]
        if s == target: return [lo, hi]
        elif s < target: lo += 1
        else: hi -= 1
    return []

# ── SAME DIRECTION ── (fast/slow pointer)
# Remove duplicates from sorted array in-place
def remove_duplicates(nums):
    if not nums: return 0
    slow = 0
    for fast in range(1, len(nums)):
        if nums[fast] != nums[slow]:
            slow += 1
            nums[slow] = nums[fast]
    return slow + 1

# ── PALINDROME CHECK ──
def is_palindrome(s):
    lo, hi = 0, len(s) - 1
    while lo < hi:
        if s[lo] != s[hi]: return False
        lo += 1; hi -= 1
    return True

# ── CONTAINER WITH MOST WATER ──
def max_area(height):
    lo, hi = 0, len(height) - 1
    max_water = 0
    while lo < hi:
        water = min(height[lo], height[hi]) * (hi - lo)
        max_water = max(max_water, water)
        if height[lo] < height[hi]: lo += 1
        else: hi -= 1
    return max_water

Sliding Window

Maintain a dynamic window over contiguous subarrays/substrings.

sliding_window.pyPython
# ── FIXED SIZE WINDOW ──
def max_sum_k(nums, k):
    window = sum(nums[:k])
    best = window
    for i in range(k, len(nums)):
        window += nums[i] - nums[i - k]   # slide: add right, remove left
        best = max(best, window)
    return best

# ── VARIABLE SIZE WINDOW ──
# Longest subarray with sum <= target
def longest_valid(nums, target):
    left = curr = best = 0
    for right in range(len(nums)):
        curr += nums[right]
        while curr > target:      # shrink when invalid
            curr -= nums[left]
            left += 1
        best = max(best, right - left + 1)
    return best

# ── LONGEST SUBSTRING WITHOUT REPEATING ──
def length_of_longest(s):
    chars = set()
    left = best = 0
    for right in range(len(s)):
        while s[right] in chars:
            chars.discard(s[left])
            left += 1
        chars.add(s[right])
        best = max(best, right - left + 1)
    return best

# ── MINIMUM WINDOW SUBSTRING ──
from collections import Counter
def min_window(s, t):
    need = Counter(t)
    missing = len(t)
    best = ""
    left = 0
    for right, ch in enumerate(s):
        if need[ch] > 0: missing -= 1
        need[ch] -= 1
        while missing == 0:            # valid window
            if not best or right - left + 1 < len(best):
                best = s[left:right + 1]
            need[s[left]] += 1
            if need[s[left]] > 0: missing += 1
            left += 1
    return best

Recursion

A function calling itself. The foundation of trees, graphs, DP, and backtracking.

The Three Laws of Recursion

1. Base case — every recursive function must have a condition where it stops.
2. Progress — every recursive call must move closer to the base case.
3. Trust the recursion — assume your function works correctly on smaller inputs. Don't trace the entire call stack in your head.

recursion.pyPython
# ── THE ANATOMY OF EVERY RECURSIVE FUNCTION ──
def recursive(problem):
    # 1. BASE CASE: stop condition
    if is_base_case(problem):
        return base_answer

    # 2. REDUCE: make the problem smaller
    smaller = reduce(problem)

    # 3. RECURSE: solve the smaller problem
    sub_answer = recursive(smaller)

    # 4. COMBINE: build answer from sub_answer
    return combine(sub_answer, problem)

# ── FACTORIAL ── (classic example)
def factorial(n):
    if n == 0: return 1         # base case
    return n * factorial(n - 1) # recurse + combine

# ── POWER FUNCTION ──
def power(base, exp):
    if exp == 0: return 1
    if exp % 2 == 0:
        half = power(base, exp // 2)
        return half * half             # fast exponentiation O(log n)
    return base * power(base, exp - 1)

# ── FLATTEN NESTED LIST ──
def flatten(lst):
    result = []
    for item in lst:
        if isinstance(item, list):
            result.extend(flatten(item))  # recurse into nested list
        else:
            result.append(item)
    return result

# ── MEMOIZATION (top-down DP) ──
# Without memo: O(2^n).  With memo: O(n).
from functools import lru_cache

@lru_cache(maxsize=None)
def fib(n):
    if n <= 1: return n
    return fib(n - 1) + fib(n - 2)

# Manual memo dict (when lru_cache isn't suitable)
memo = {}
def fib_manual(n):
    if n in memo: return memo[n]
    if n <= 1: return n
    memo[n] = fib_manual(n-1) + fib_manual(n-2)
    return memo[n]

# ── RECURSIVE vs ITERATIVE CONVERSION ──
# Any recursion can be converted to iteration using an explicit stack

# Recursive DFS
def dfs_recursive(node, visited):
    visited.add(node)
    for nei in graph[node]:
        if nei not in visited:
            dfs_recursive(nei, visited)

# Equivalent iterative DFS (avoids Python recursion limit)
def dfs_iterative(start):
    visited, stack = set(), [start]
    while stack:
        node = stack.pop()
        if node in visited: continue
        visited.add(node)
        stack.extend(nei for nei in graph[node] if nei not in visited)

# ── INCREASE RECURSION LIMIT ──

Recursion Debugging Tips

  • Draw the call tree for small inputs (n=3, n=4). Visualizing the tree makes recursion click immediately.
  • Trust the function — when writing recursion, assume your function returns the correct answer for smaller inputs. Don't try to trace every call.
  • Hit recursion limit? Either switch to iterative with an explicit stack, or add sys.setrecursionlimit().
  • Seeing repeated subproblems? Add @lru_cache — you just converted recursion into DP.

Dynamic Programming

The hardest topic — and the most rewarding. Break problems into overlapping subproblems.

The DP Framework — Use This Every Time

Step 1: Define the subproblem — what does dp[i] represent?
Step 2: Find the recurrence — how does dp[i] relate to smaller subproblems?
Step 3: Identify base cases — what are the trivial answers?
Step 4: Determine order — what do you need to compute before dp[i]?
Step 5: Extract answer — where in dp is your final answer?

dynamic_programming.pyPython
from functools import lru_cache

# ── FIBONACCI (the hello world of DP) ──
# Naive: O(2^n)   Memoized: O(n)
@lru_cache(maxsize=None)
def fib(n):
    if n <= 1: return n
    return fib(n-1) + fib(n-2)

# ── CLIMBING STAIRS ──
# dp[i] = ways to reach step i
def climb_stairs(n):
    if n <= 2: return n
    a, b = 1, 2
    for _ in range(3, n + 1):
        a, b = b, a + b
    return b

# ── 0/1 KNAPSACK ──
# dp[i][w] = max value with first i items and capacity w
def knapsack(weights, values, capacity):
    n = len(weights)
    dp = [[0] * (capacity + 1) for _ in range(n + 1)]
    for i in range(1, n + 1):
        for w in range(capacity + 1):
            dp[i][w] = dp[i-1][w]
            if weights[i-1] <= w:
                dp[i][w] = max(dp[i][w],
                               dp[i-1][w - weights[i-1]] + values[i-1])
    return dp[n][capacity]

# ── LONGEST COMMON SUBSEQUENCE ──
def lcs(s1, s2):
    m, n = len(s1), len(s2)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s1[i-1] == s2[j-1]: dp[i][j] = dp[i-1][j-1] + 1
            else: dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

# ── COIN CHANGE ──
def coin_change(coins, amount):
    dp = [float('inf')] * (amount + 1)
    dp[0] = 0
    for coin in coins:
        for x in range(coin, amount + 1):
            dp[x] = min(dp[x], dp[x - coin] + 1)
    return dp[amount] if dp[amount] != float('inf') else -1

# ── LONGEST INCREASING SUBSEQUENCE ──
def lis(nums):
    dp = [1] * len(nums)
    for i in range(1, len(nums)):
        for j in range(i):
            if nums[j] < nums[i]:
                dp[i] = max(dp[i], dp[j] + 1)
    return max(dp)

Greedy Algorithms

Make the locally optimal choice at each step. Simple and powerful — when it works.

When Does Greedy Work?

Greedy works when a problem has the greedy choice property — the locally best choice leads to a globally best solution. It does NOT always work (that's when you need DP). The key question to ask: "Will choosing the best option right now ever hurt me later?" If no, greedy is valid.

greedy.pyPython
# ── ACTIVITY SELECTION (classic greedy) ──
# Choose maximum non-overlapping intervals
# Greedy: always pick the interval that ends earliest
def max_activities(intervals):
    intervals.sort(key=lambda x: x[1])  # sort by end time
    count = 1
    end = intervals[0][1]
    for start, finish in intervals[1:]:
        if start >= end:       # no overlap
            count += 1
            end = finish
    return count

# ── MINIMUM INTERVALS TO REMOVE ──
# (LeetCode 435 — Non-overlapping Intervals)
def erase_overlap_intervals(intervals):
    intervals.sort(key=lambda x: x[1])
    removed = 0
    end = float('-inf')
    for s, e in intervals:
        if s >= end:
            end = e
        else:
            removed += 1        # overlaps — remove it
    return removed

# ── JUMP GAME ── (can you reach the end?)
def can_jump(nums):
    max_reach = 0
    for i, jump in enumerate(nums):
        if i > max_reach: return False   # can't reach here
        max_reach = max(max_reach, i + jump)
    return True

# ── MINIMUM NUMBER OF JUMPS ──
def jump(nums):
    jumps = curr_end = curr_far = 0
    for i in range(len(nums) - 1):
        curr_far = max(curr_far, i + nums[i])
        if i == curr_end:         # reached the boundary of current jump
            jumps += 1
            curr_end = curr_far
    return jumps

# ── MEETING ROOMS II ── (minimum rooms needed)
import heapq
def min_meeting_rooms(intervals):
    intervals.sort(key=lambda x: x[0])  # sort by start
    heap = []   # min-heap of end times
    for start, end in intervals:
        if heap and heap[0] <= start:
            heapq.heapreplace(heap, end)   # reuse room
        else:
            heapq.heappush(heap, end)      # need new room
    return len(heap)

# ── GAS STATION ──
def can_complete_circuit(gas, cost):
    total = curr = start = 0
    for i in range(len(gas)):
        diff = gas[i] - cost[i]
        total += diff
        curr  += diff
        if curr < 0:          # can't reach here from current start
            start = i + 1    # try next station as start
            curr = 0
    return start if total >= 0 else -1

Greedy vs DP — How to Decide?

  • Try greedy first — if you can prove the greedy choice never hurts, use it. It's simpler and faster.
  • If greedy fails on a counterexample, switch to DP.
  • Common greedy patterns: sort by start/end time (interval problems), always take the max/min available, move a pointer only when forced.
  • Prove it: "If I pick X greedily here, can the optimal solution NOT pick X?" If it must pick X, greedy is correct.

Backtracking

Explore all possibilities, prune dead ends early. The brute force of smart coders.

backtracking.pyPython
# ── UNIVERSAL BACKTRACKING TEMPLATE ──
def backtrack(state, choices):
    if is_solution(state):
        result.append(list(state))   # copy state!
        return
    for choice in choices:
        if is_valid(choice, state):
            state.append(choice)         # make choice
            backtrack(state, choices)   # recurse
            state.pop()                  # undo choice ← THE KEY

# ── PERMUTATIONS ──
def permutations(nums):
    result = []
    def backtrack(path, remaining):
        if not remaining:
            result.append(path[:])
            return
        for i, n in enumerate(remaining):
            path.append(n)
            backtrack(path, remaining[:i] + remaining[i+1:])
            path.pop()
    backtrack([], nums)
    return result

# ── SUBSETS ──
def subsets(nums):
    result = []
    def backtrack(start, path):
        result.append(path[:])
        for i in range(start, len(nums)):
            path.append(nums[i])
            backtrack(i + 1, path)   # i+1 to avoid reuse
            path.pop()
    backtrack(0, [])
    return result

# ── N-QUEENS ──
def solve_n_queens(n):
    result = []
    cols = set(); pos_diag = set(); neg_diag = set()
    board = ["." * n for _ in range(n)]
    def backtrack(row):
        if row == n:
            result.append(board[:])
            return
        for col in range(n):
            if col in cols or (row+col) in pos_diag or (row-col) in neg_diag:
                continue
            cols.add(col); pos_diag.add(row+col); neg_diag.add(row-col)
            board[row] = "."*col + "Q" + "."*(n-col-1)
            backtrack(row + 1)
            cols.discard(col); pos_diag.discard(row+col); neg_diag.discard(row-col)
            board[row] = "." * n
    backtrack(0)
    return result

Graph Algorithms

Shortest paths, spanning trees, and ordering — the big guns.

graph_algorithms.pyPython
import heapq
from collections import defaultdict, deque

# ── DIJKSTRA (Shortest Path — weighted graph) ──
# graph: {node: [(weight, neighbor), ...]}
def dijkstra(graph, start):
    dist = {node: float('inf') for node in graph}
    dist[start] = 0
    pq = [(0, start)]
    while pq:
        d, u = heapq.heappop(pq)
        if d > dist[u]: continue      # stale entry, skip
        for w, v in graph[u]:
            nd = d + w
            if nd < dist[v]:
                dist[v] = nd
                heapq.heappush(pq, (nd, v))
    return dist

# ── TOPOLOGICAL SORT (Kahn's BFS algorithm) ──
# For directed acyclic graphs (DAGs)
def topo_sort(n, edges):
    adj = defaultdict(list)
    indegree = [0] * n
    for u, v in edges:
        adj[u].append(v)
        indegree[v] += 1
    q = deque([i for i in range(n) if indegree[i] == 0])
    order = []
    while q:
        node = q.popleft()
        order.append(node)
        for nei in adj[node]:
            indegree[nei] -= 1
            if indegree[nei] == 0:
                q.append(nei)
    return order if len(order) == n else []   # [] if cycle

Union Find (DSU)

Near O(1) grouping and connectivity queries. Essential for graph problems.

union_find.pyPython
class DSU:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n
        self.count = n       # number of components

    def find(self, x):
        # Path compression: flatten tree on find
        if self.parent[x] != x:
            self.parent[x] = self.find(self.parent[x])
        return self.parent[x]

    def union(self, x, y):
        px, py = self.find(x), self.find(y)
        if px == py: return False  # already same component
        # Union by rank: attach smaller tree under larger
        if self.rank[px] < self.rank[py]: px, py = py, px
        self.parent[py] = px
        if self.rank[px] == self.rank[py]: self.rank[px] += 1
        self.count -= 1
        return True

    def connected(self, x, y):
        return self.find(x) == self.find(y)

# Example: count connected components
dsu = DSU(5)
dsu.union(0, 1); dsu.union(2, 3)
print(dsu.count)  # 3 (components: {0,1}, {2,3}, {4})

# XOR all numbers — duplicates cancel out
def single_number(nums):
    res = 0
    for n in nums: res ^= n
    return res

# ── COUNT SET BITS (Hamming weight) ──
def hamming_weight(n):
    count = 0
    while n:
        n &= (n - 1)   # clears lowest set bit
        count += 1
    return count
